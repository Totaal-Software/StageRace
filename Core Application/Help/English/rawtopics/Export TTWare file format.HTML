Export Competitors for Skunkware TT*Ware

Competitors of time trials can be exported from StageRace so they can be used in Skunkware TT*Ware.

In contrary to the other supported import/export formats, TT*Ware is specifically applicable to time trials.

Note that you can also import TT*Ware files for ranking a stage.

Introduction

Broder's Skunkware (U.S.A.) develops custom IP-enabled, client-server scoring and timing systems and broadcast graphics solutions for live and live-to-tape sporting events.

Their TT*Ware software allows for real-time support during time trials.

In cooperation with Phoenix Sports Technology from Pennsylvania, U.S.A., the StageRace program has been made capable to import files from TT*Ware.

Exporting

Within TT*Ware text file import is possible (Import Comma-Delimited Data into a Race).

In order to have StageRace generating such output files, the Export competitors for Skunkware TT*Ware option in the Tools menu has to be chosen with the required time trial being selected. Note that TT*Ware files can only be exported for one stage at a time.

The location of the files generated depends on the path set in the Options dialog.

For TT*Ware operation, please refer to the appropriate documentation.

Output is generated in the form of a so-called comma separated file of which the file name is set up as follows:

[Race Title] - Competitors list [FS]-[HS] (TTWare).csv

For instance:

Ronde van Nederland - Competitors list 3-1 (TTWare).csv

File Set-up

Every line in the file consists of one competitor with the following comma-delimited fields:

  • Time trial start time
  • Start no.
  • Full name
  • Team code
  • Team name
  • Country
  • Category
  • License
  • UCI code

Note that while TT*Ware differentiates between first and last name, StageRace can only provide the full name of a competitor.

A good possible Import Parameters setting within TT*Ware for this could be:

  • Last Name: 3
  • NAT: 6
  • Team: 4 or 5
  • Start number: 2
  • Bib Number: 2
  • Start Time, Run 1: 1
  • UCI Number: 9
  • Class: 7

(Other values left to zero)

Note that testing with v3.07 of TT*Ware an error occurs (saying "Import file is Binary, cannot continue") when in the file non-ANSI characters are used, such as é, à, ö, ß, etc. If you want to use file export for TT*Ware, you should avoid this kind of characters in competitor and team names and the like.
